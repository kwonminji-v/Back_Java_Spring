서블릿 (servelet)은 서버에서 웹페이지 등을 동적으로 생성하거나
데이터 처리를 수행하기 위해 자바로 작성된 프로그램입니다.
servlet은 자바코드 안에 HTML 태크가 삽입되며 자바언어로 되어있으며, .java가 확장자 입니다.
서블릿의 단어는 Server + Let의 합성어라고 알려지기도 했고,
Servet + Applet으로 알고 있는 사람도 있습니다.
즉) 쉽게 말하면 '클라이언트의 요청을 처리하고 그 결과를 다시 클라이언트에게 전송하는
servlet 클래스의 구현 규칙을 지킨 자바프로그램' 인 것 입니다.
서블릿은 자바언어를 웹 어플리케이션에 조금 더 개발하기 쉽게 하기 위해서 만든 API (라이브러리, 클래스 들)이며,
규약에 맞는 라이브러리나 클래스들을 상속 및 구현하여 만든 클래스들을 서블릿이라고 합니다.

Servelet : java 클래스 형태의 웹 어플리케이션, 수행속도가 빠른 장점이 있습니다.
자바에서 html 등의 스크립트 언어를 사용할 수 있게 합니다.

데이터 전송 방식
get : 조소창을 타고 넘어가기 때문에 서버로 보내는 데이터를 사용합니다. 
보안에 취약, 255자 (1 byte) 이하의 적은 용량 데이터를 전송 
post : html header를 타고 넘어가기 때문에 보안에 강합니다. 255자 이상의 대용량 데이터를 전송
handle : 두 가지 방식을 모두 활용합니다.


Jsp
HTML을 코딩하기 너무 불편해서 HTML 내부에 Java 코드를 삽입하는 형식이 JSP입니다.
다시 말해 서블릿의 단점을 보완하고자 만든 서블릿 기반의 스크립트 기술입니다.
JSP(Java Server Pages)는 서블릿 기반의 '서버 스크립트 기술'입니다.
정적 데이터만 처리할 수 있는 HTML의 단점을 극복한 스크립트 언어입니다.

<배포 서술자 - web.xml>
웹 애플리케이션의 환경설정 정보를 담은 파일입니다.
WAS가 구동될 때 처음 이 파일을 읽어 설정 내용을 톰캣에 적용하게 됩니다.
서블릿 설정, 필터설정, 웰컴파일목록, 오류페이지 처리와 같은 설정을 할 수 있습니다.

WAS(Web Application Server) ... 톰캣
JSP, ASP, PHP 등 사용자의 입력을 받아 서버에게 무언가 처리하고 그 결과를 보여주는 동적인 데이터를 처리하는 방식입니다.
WAS는 규모가 크고 엔터프라이즈 환경에 필요한 트랜잭션, 보안 ,트래픽 관리, DB 커넥션 풀
사용자 관리 등등 다양하고 강력한 기능을 제공하는 S/W를 의미합니다.

응답코드 
: 서버 응답 코드는 클라이언트의 REQUEST를 처리한 후 상태에 따라 반환하는 코드입니다.
  응답 코드는 크게 5가지로 분류됩니다.
  
 1XX: Informational(정보 제공)  / 102 : Processing
임시 응답으로 현재 클라이언트의 요청까지는 처리되었으니 계속 진행하라는 의미입니다. 
HTTP 1.1 버전부터 추가되었습니다.

2XX: Success(성공) / 201 : 클라이언트가 요청한 정보를 정상적으로 저장 완료
클라이언트의 요청이 서버에서 성공적으로 처리되었다는 의미입니다

3XX: Redirection(리다이렉션)
완전한 처리를 위해서 추가 동작이 필요한 경우입니다. 주로 서버의 주소 또는 요청한 URI의 웹 문서가 이동되었으니 그 주소로 다시 시도하라는 의미입니다.

4XX: Client Error(클라이언트 에러) / 404 : 찾을 수 없음 . 페이지 접속할 수 없을 때 반환
없는 페이지를 요청하는 등 클라이언트의 요청 메시지 내용이 잘못된 경우를 의미합니다.

5XX: Server Error(서버 에러) / 500 : 내부 서버 오류 (톰캣 에러)
서버 사정으로 메시지 처리에 문제가 발생한 경우입니다. 서버의 부하, DB 처리 과정 오류, 서버에서 익셉션이 발생하는 경우를 의미합니다.

Context Path (컨텍스트 패스)
서블릿을 요청하기 위한 URL
JSP나 Servelet을 실행하면 항상 앞에 생성되는 http://localhost는 웹 서버에 접속하기 위한 ip 주소
:8080은 톰캣에서 최초 지정된 포트번호입니다. 그 뒤에 오는 것을 컨텍스트 패스라고 합니다.
이클립스와 톰캣을 연동해 웹 애플리케이션을 실행할 때 server.xml 자동으로 추가됩니다.
